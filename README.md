Farmacia Lily — Smart Inventory System
This project was born from a real need. I'm part of a family-run pharmacy operated by four people, where inventory management relied more on habit than data for years.
Everything changed when I discovered artificial intelligence. That moment led me to rethink my future and decide to build a career in programming, AI and software architecture — starting by solving a real problem in my own environment.
The Problem
We are a low-traffic pharmacy, but with high-value sales. Losing a single sale can represent an entire day's revenue.
The main challenges were:

No clear process for making purchases
Decisions based on habit rather than data
Excess of some products and constant shortage of others
No reliable sales history
Team resistance to adopting new technology

The Solution
The goal was not to impose technology, but to design it around the people who would use it.
Inspired by mobile OCR and systems like Amazon Go, I developed a system where:

Any team member can register products from their phone
Data is stored in the cloud, centralizing information
Daily sales can be captured via photo
Incoming inventory is logged with images
Smart restock alerts are generated, both scheduled and urgent
Each product has a configurable minimum stock based on its importance

Current State
The system currently allows:

Registering products through a web interface
Storing data in the cloud (Firebase Firestore)
Querying inventory in real time
Detecting low-stock products

This represents the first functional MVP, with real potential for use in the business.
Next Steps

Automated sales module
OCR integration for image-based capture
Weekly purchase report generation
Improved user experience
